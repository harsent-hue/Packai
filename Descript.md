# Демо: полный цикл от анализа до архитектуры

Давайте пройдём полный путь — от проверки окружения до генерации готового промпта для кодогенерации. Это покажет, как Unified Pip Manager + ИИ-Архитектор превращается из простого менеджера пакетов в настоящего помощника разработчика.

## Шаг 1: Запуск и автоматическое сканирование

После запуска приложение сразу начинает анализировать выбранное Python-окружение. Оно запускает `pip list --format=json` и `pip list --outdated --format=json`, чтобы построить полную картину установленных и устаревших пакетов.

**Рис. 1. Интерфейс при запуске — автоматическое сканирование пакетов**

На первом скриншоте видно главное окно с вкладками, строкой интерпретатора и процессом загрузки. Приложение определило, что используется .venv, и уже начало сбор данных.

> Это важный момент: вы не жмёте «Обновить» — система делает это автоматически. Всё ради удобства и актуальности информации.

## Шаг 2: Обнаружение устаревших пакетов

Через несколько секунд таблица заполняется. Мы видим список библиотек, их версии и статус. Некоторые помечены как «Устарел» — например, click, requests, cryptography.

**Рис. 2. Список установленных пакетов с индикацией устаревших**

Таблица использует чёткое цветовое оформление (тёмная тема customtkinter), чекбоксы для выбора и лог внизу для операций. Пользователь может быстро оценить состояние проекта.

> Здесь уже можно принимать решения: обновлять всё сразу или выбрать только критичные зависимости.

## Шаг 3: Целевое обновление пакета

Мы отмечаем галочкой click — достаточно кликнуть по пустой ячейке в колонке "Выбор". Затем нажимаем "Обновить выбранные пакеты".

Сразу активируется лог в реальном времени: каждая строка вывода `pip install --upgrade click` появляется мгновенно.

**Рис. 3. Лог обновления в реальном времени**

Видно, как pip скачивает новую версию, разрешает зависимости и завершает установку. В конце — сообщение: ✅ click обновлён успешно.

> Всё это происходит в фоновом потоке, интерфейс не зависает. Это достигнуто через threading, что особенно важно при обновлении тяжёлых пакетов.

## Шаг 4: Авто-категоризация с помощью ИИ

Переходим на вкладку "📂 Категории". Здесь можно редактировать группы вручную, но нас интересует кнопка "🧠 Авто-категоризация (LLM)".

При нажатии приложение отправляет список всех установленных пакетов в LLM (например, llama3.2 через Ollama) с промптом:

```
«Сгруппируй эти библиотеки по логическим категориям: веб, ML, безопасность и т.д. Верни JSON с категориями и списками»
```

Через 10–20 секунд ИИ возвращает структурированный ответ — и категории автоматически обновляются.

**Рис. 4. Новые категории, сгенерированные ИИ**

В таблице появились такие группы, как "Multi-Agent Orchestration", "Industrial Crawling", "OCR", "RAG Framework" — не просто общие названия, а осмысленные домены, отражающие реальное назначение стека.

> Это мощный инструмент для анализа legacy-проектов, когда вы только начинаете разбираться в кодовой базе.

## Шаг 5: Генерация архитектуры по задаче

Теперь перейдём к главному — проектированию.

На вкладке "🤖 ИИ-Архитектор" вводим сложную задачу:

```
«Мне нужен конвейер: поиск в сети, парсинг сайтов, распознавание зависимых ссылок с возможностью настроить глубину слоёв (например книга, глава №, раздел№, подраздел№), сперва нужно показать логику и спросить, то ли он нашёл, что исключить, а что парсить. Распознавание графиков на картинках и сохранение структуры в мультимодальный RAG с возможностью поиска по изображениям и тексту.»
```

Нажимаем "🚀 Анализировать и подобрать стек".

## Шаг 6: Получение вариантов и финального промпта

Через некоторое время ИИ возвращает структурированный JSON с несколькими вариантами архитектуры. Мы выбираем один из них — например, тот, что использует Scrapy для глубокого парсинга и CrewAI для оркестрации.

Приложение автоматически строит итоговый промпт, включающий:

- Исходную задачу
- Рекомендованный стек
- Категории и обоснование
- Полный список уже установленных пакетов (чтобы минимизировать дублирование)

**Рис. 5. Итоговый промпт для кодогенерации**

```
# Исходная задача:
Мне нужен конвейер: поиск в сети, парсинг сайтов, распознование зависимых сслок с возможностью настроить глубину слоёв (например книга, глава №, раздел№, подраздел№), сперва нужно показать логику и спросить, то ли он нашёл, что исключить, а что парсить. распознавание графиков на картинках и сохранение структуры в мультимодальный RAG с возможностью поиска по изображениям и тексту.

# Рекомендованный стек технологий (ИИ-архитектор):
Библиотеки: crewai, scrapy, trafilatura, pytesseract, llama-index, chromadb, fastapi
Категории: Multi-Agent Orchestration, Industrial Crawling, OCR, RAG Framework
Обоснование: Scrapy — лучший выбор для глубокого рекурсивного парсинга с настраиваемой глубиной. CrewAI организует процесс взаимодействия (поиск -> проверка -> сбор). LlamaIndex предоставляет продвинутые инструменты для индексации иерархических структур (книга/глава) в связке с ChromaDB.

# Текущее окружение разработчика:
В системе уже установлены: aiohappyeyeballs, aiohttp, aiosignal, altair, annotated-doc, ... [полный список]

# Задание для кодогенерации:
Напиши готовый к запуску код на Python, решающий поставленную задачу с использованием рекомендованного стека. Код должен быть модульным, с обработкой ошибок, без заглушек.
```

> **Почему это важно?**
> Обычно разработчик:
> - Сам ищет библиотеки
> - Вручную составляет промпт
> - Не учитывает уже установленные зависимости
> 
> А здесь:
> - ИИ знает ваш стек
> - Учитывает реальное окружение
> - Предлагает совместимые решения
> - Генерирует рабочий промпт за 20 секунд
> 
> Это следующий уровень инструментария — не просто GUI для pip, а интеллектуальный ассистент разработки.

- Экспортировать requirements.txt с актуальными версиями
- Продолжить работу — с чистой головой и чётким планом
